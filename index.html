Documentação Interface Mapeamento de campos Estrutura JSON – Ticket HugMe
Esta documentação descreve a integração entre Salesforce e o Hugme (plataforma ReclameAQUI), permitindo sincronizar tickets de reclamação, empresas e dados de atendimento entre os sistemas.

Visão geral

A integração Salesforce x Hugme permite:

  • Consultar e listar tickets de reclamação no Salesforce (via API Hugme).
  • Visualizar detalhes de tickets, empresas (companies) e reputação.
  • Utilizar os endpoints da API ReclameAQUI (Hugme) a partir de flows, Apex ou conectores no Salesforce.

Requisitos

  • Conta e credenciais de acesso à API ReclameAQUI (Hugme).
  • Salesforce (Sales Cloud ou Service Cloud) com permissão para configurar integrações (Callouts / Named Credentials ou Connected App).
  • URL base da API e endpoints documentados (ver links abaixo).

Configuração no Salesforce

1. Named Credential (recomendado)

Crie uma Named Credential apontando para a URL base da API Hugme. Configure autenticação (por exemplo, OAuth ou usuário/senha) conforme fornecido pelo ReclameAQUI.

2. Chamadas à API

Use a Named Credential como host nas chamadas HTTP (Apex HttpRequest / Http ou Flow “Call external system”). Exemplo de endpoints:

RecursoDescrição
Retrieve all TicketsLista de tickets com filtros (ex.: data de modificação).
Retrieve Ticket by idDetalhes de um ticket específico.
List Companies / Search CompaniesEmpresas e busca.
Retrieve ReputationDados de reputação.

Fluxo de dados (resumo)

Salesforce → (HTTPS) → API ReclameAQUI (Hugme) → resposta JSON (tickets, companies, meta, etc.). A estrutura completa da resposta de tickets está documentada na página “Estrutura JSON – Ticket HugMe”.
